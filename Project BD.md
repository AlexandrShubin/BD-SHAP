# 2. Назначение и цели создания системы

## 2.1. Назначение системы

> Автоматизация предполагает создание единого информационного ресурса о транспортных средствах судоходных компаний в целях обеспечения транспортной безопасности субъектов транспортной инфраструктуры в соответствии с действующим законодательством.

> Проектом предусмотрено централизация данных о транспортных средствах разных структурных подразделений транспортных компаний (кадры, финансовый отдел, транспортный отдел, отдел обеспечения, служба безопасности мореплавания, отдел транспортной безопасности).

> База данных предназначена для повышения оперативности и качества принимаемых управленческих решений сотрудниками Заказчика.
> Основным назначением Базы данных является автоматизация основных показателей о транспортных средствах в деятельности Заказчика.

## 2.2. Цели создания системы

> База данных создается с целью:
> - обеспечения сбора и первичной обработки исходной информации, необходимой для принятия управленческих решений и обеспечения транспортной безопасности;
> - создания единой системы данных по транспортным средствам;

> - повышения качества (полноты, точности, достоверности, своевременности, согласованности) информации;

> В результате создания хранилища данных должны быть улучшены значения следующих показателей:
> - время сбора и первичной обработки исходной информации;
> - количество информационных систем, используемых для подготовки аналитической отчетности;
> - время, затрачиваемое на подготовку информации.

# 3. Характеристика объектов автоматизации

> Создание базы данных необходимо для транспортного отдела, технической службы, службы по безопасности мореплавания, отдела транспортной безопасности.

> Объектом автоматизации являются объекты и бизнес-процессы, выполняемые в структурных подразделениях Заказчика. Объектами автоматизации будут являться бизнес-процессы, выполняемые в структурных подразделениях Заказчика.

## 5. Состав и содержание работ по созданию системы

> Работы по созданию системы выполняются в три этапа:
Проектирование. Разработка эскизного проекта. Разработка технического проекта (продолжительность - 1 месяц).
> Разработка рабочей документации. Адаптация программ (продолжительность - 3 месяца).
> Ввод в действие (продолжительность – 1 месяц).
